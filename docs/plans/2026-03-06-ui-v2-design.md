# UI V2 Design — Chat-Centric Single Page

**Goal:** Replace the tab-based Streamlit UI with a chat-first single-page layout optimized for LevFin analysts who primarily use Q&A to extract deal terms from credit agreements.

**Users:** Investment banking analysts, RBC Leveraged Finance. Primary workflow is uploading one agreement at a time and asking targeted questions about provisions. They copy answers into memos/models and screen-share the tool in meetings.

**Design principles:** Clean minimalism, progressive disclosure, speed/responsiveness cues. No emojis.

## Layout

No tabs. Three zones: header bar, collapsible sidebar, main chat area.

### Header Bar
- Slim (48-52px), dark navy background, white text
- Left: "Credit Agreement Analyzer" text wordmark (DM Sans 600)
- Right cluster: New Chat (ghost button), Definitions (ghost button + SVG search icon, opens modal), document name badge/pill, Generate Report (primary button, disabled when no doc)
- All buttons compact (13-14px font)

### Sidebar (Collapsible, Navy Gradient)
Three sections separated by subtle dividers:

1. **Model Status** — Compact rail card: provider + model + status dot (green/amber). Retry button only when unhealthy.
2. **Document** — File uploader, Index PDF button. When indexed: compact 2x2 stats grid (pages, sections, chunks, definitions as small numbers). Remove Document as subtle text link. Indexing progress renders here as step pipeline.
3. **About** — "RBC Leveraged Finance" caption + model version muted text.

### Main Chat Area

**State 1 (No document):** Centered message "Upload a credit agreement to get started" with subtle pointer toward sidebar.

**State 2 (Document loaded, no messages):** Suggested question cards in 3x2 grid. Compact rounded rectangles, hover lift. Caption "Or type your own question below." Sticky chat input at bottom.

**State 3 (Active conversation):** Message stream. User messages right-aligned, subtle blue tint. Assistant messages left-aligned, white background with:
- Answer text (streaming with cursor during generation)
- Section-awareness status during generation: "Searching 5 sections..." -> "Reading Section 4.01, 7.03..." -> "Composing answer..."
- Key-value formatting for structured data (reuses format_report_body LABEL: VALUE detection)
- Context strip below answer: `HIGH confidence · 5 chunks · Section 4.01, 7.03 · 1.2s` — click to expand full chunks table
- Copy button on hover (top-right)
- Subtle timestamp ("2:34 PM") on each message

"Suggestions" chip near chat input to re-access suggested questions.

## Definitions Search Modal
- `@st.dialog` overlay with backdrop
- Auto-focus search input, filter-as-you-type
- Term name bold blue, definition preview ~2 lines
- Click term to expand full definition inline
- Result count at bottom
- Close with X or backdrop click
- Show first 30 results, scrollable

## Report Drawer
- `@st.dialog(width="large")` overlay
- Header: borrower name, date subtitle, stats pills, Download PDF button
- Two-column: left quick-nav with status dots, right section cards
- **Progressive rendering:** All 10 section cards appear immediately as skeleton shells. Each fills in as generation completes. Section 1 first (sequential), sections 2-10 fill progressively (parallel workers).
- Quick-nav dots: green (complete), pulsing blue (generating), gray (pending)
- Per-section "Retry" link on LOW confidence or error sections
- Report cached — reopening dialog shows existing report instantly

## Indexing Progress (Sidebar Step Pipeline)
```
  checkmark  Extracting text         42 pages
  checkmark  Detecting structure     18 sections
  checkmark  Parsing definitions     127 terms
  (active)   Building embeddings     64/203 chunks
  (pending)  Creating search index
```
- Complete: checkmark, muted text, result count right
- Active: filled dot, subtle pulse animation, live count
- Pending: hollow dot, dimmed text
- Thin overall progress bar above steps
- Collapses to compact stats grid on completion

## Style
- Existing RBC palette: navy (#001A3E), blue (#0051A5), gold (#C8A000/#D4AF37), ink (#0F1A2E)
- Typography: DM Sans headings, Source Sans 3 body
- No emojis anywhere
- Smooth transitions (0.2s ease) on interactive elements
- Skeleton loaders for progressive content
